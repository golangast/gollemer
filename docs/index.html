Multi-Orchestrator with NLP Understanding

Your smart assistant for code generation.

Overview

The Multi-Orchestrator has been enhanced with semantic understanding using a Mixture of Experts (MoE) model and Named Entity Recognition (NER) tagging. Instead of relying on simple keyword matching, it now understands what you're writing in natural language.

Before vs. After

Before (Keyword-based)

The orchestrator would only recognize specific keywords:

  • "create webserver"
  • "create handler"
  • "create database"

After (Semantic Understanding)

Now you can write naturally, and the orchestrator will understand your intent:

  • "I want to build a webserver with an authentication handler"
  • "Make me a Go server that has JWT support"
  • "Create an API with a database for storing user data"

How It Works

The enhanced orchestrator uses a multi-stage NLP pipeline:

  1. Intent Classification - Determines what you want to do (create, modify, delete, etc.)
  2. Named Entity Recognition (NER) - Extracts key entities like handler names, database names, component names
  3. Semantic Parsing - Builds a structured representation of your command
  4. Template Filling - Generates the appropriate code based on the semantic understanding

Features

  • ✅ Natural Language Understanding - Write commands as you would speak them
  • ✅ Intent Detection - Automatically determines what you want to do
  • ✅ Entity Extraction - Intelligently extracts names, types, and parameters
  • ✅ Fallback Support - Falls back to keyword matching if semantic parsing fails
  • ✅ Semantic Output - Generates structured JSON representation of commands
  • ✅ Git Integration - Tracks changes with automatic commits
  • ✅ History & Revert - View command history and revert to previous states

Additional Tools

The project includes a suite of powerful tools for training models and interacting with the system in new ways.

  • Model Training: Easily train the Word2Vec, Mixture of Experts (MoE), and intent classification models.
  • MoE Inference: Run inference directly with the MoE model to test its responses.
  • Interactive LLM Mode: A chat-like interface for performing file system operations and asking questions about your project.

Example Usage

# Run the multi-orchestrator
go run ./cmd/multi_orchestrator

# --- Or run additional tools ---

# Train the Word2Vec model
go run main.go -train-word2vec

# Train the MoE model
go run main.go -train-moe

# Run in interactive LLM mode
go run main.go -llm

# Inside LLM mode, you can type commands like:
> create file my_new_file.txt
> list files
> what is in README.md
